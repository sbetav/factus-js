El paquete `@factus-js/constants` expone objetos de constantes con tipado estricto para todos los códigos DIAN y valores enumerados utilizados en la API de Factus. Usar estas constantes en lugar de valores literales mejora la legibilidad del código y evita errores tipográficos.

## Instalación

Si instalaste `factus-js`, las constantes ya están disponibles — no necesitas instalar nada más:

```ts
import { PaymentFormCode } from "factus-js";
```

Si usas los paquetes internos directamente, instala `@factus-js/constants` por separado:

<Tabs items={['npm', 'yarn', 'pnpm', 'bun']}>

<Tab value="npm">

```bash
npm install @factus-js/constants
```

</Tab>

<Tab value="yarn">

```bash
yarn add @factus-js/constants
```

</Tab>

<Tab value="pnpm">

```bash
pnpm add @factus-js/constants
```

</Tab>

<Tab value="bun">

```bash
bun add @factus-js/constants
```

</Tab>

</Tabs>

## Uso

```ts
import {
  InvoiceDocumentType,
  PaymentFormCode,
  PaymentMethodCode,
} from "factus-js"; // o desde "@factus-js/constants"

console.log(PaymentFormCode.CASH_PAYMENT.value); // "1"
console.log(PaymentFormCode.CASH_PAYMENT.description); // "Pago de contado"
```

<Callout type="info">
  Cada entrada de constante expone dos propiedades: `value` (el código DIAN) y `description` (la etiqueta legible por humanos). El tipo alias derivado (p. ej. `PaymentFormCode`) sigue resolviendo a la unión de literales válidos de `value`, por lo que puedes usarlo directamente como restricción de tipo en tus propias funciones: `(code: PaymentFormCode) => void`.
</Callout>
