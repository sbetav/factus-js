import { Callout } from "fumadocs-ui/components/callout";

## Firma del método

```ts
emitRadianEvent(
  number: string,
  eventType: EventCode,
  input: RadianEventUpdateInput,
): Promise<RadianEventUpdateResponse>
```

## Endpoint Factus

`POST /v1/bills/radian/events/update/{number}/{event_type}`

Fuente oficial: [Aceptación tacita](https://developers.factus.com.co/facturas/aceptacion-tacita/index.html)

## Tipos de evento (`EventCode`)

- `ReceiptAcknowledgement` (`"030"`)
- `InvoiceClaim` (`"031"`)
- `GoodsOrServiceReceipt` (`"032"`)
- `ExpressAcceptance` (`"033"`)
- `TacitAcceptance` (`"034"`)

## Ejemplo

```ts
import { EventCode, IdentityDocumentTypeId } from "factus-js";

await factus.bills.emitRadianEvent("SETP990000049", EventCode.TacitAcceptance, {
  identification_document_code: IdentityDocumentTypeId.CitizenshipId,
  identification: "12345667",
  first_name: "Pepito",
  last_name: "Perez",
  job_title: "Desarrollador de software",
  organization_department: "Sistemas",
});
```

<Callout type="info">
  Este método cubre el flujo documentado por Factus para emisión de eventos
  RADIAN (incluyendo aceptación tacita, según el `eventType`).
</Callout>

<Callout type="warn">
  Es probable que este método no funcione en entorno sandbox.
</Callout>
