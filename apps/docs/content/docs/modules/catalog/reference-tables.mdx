Esta guía consolida las tablas oficiales de Factus y su equivalente en
constantes de `factus-js`.

Fuente oficial:
[Tablas de referencia](https://developers.factus.com.co/v1/tablas-de-referencia/tablas/index.html)

## Modo de uso

Cada tabla de referencia se expone desde la raíz del paquete y sigue el mismo
patrón:

- **Mapa de constantes** — clave-valor para usar directamente en los campos de la API.
- **Mapa `*Info`** — metadatos legibles. Siempre incluye `description` y, en algunos casos (como documentos de identidad), también `abbreviation`.

### Ejemplo

```ts
import { IdentityDocumentTypeId, IdentityDocumentTypeIdInfo } from "factus-js";

// Usar un código en un payload de la API
const docType = IdentityDocumentTypeId.CitizenshipId; // "3"

// Descripción y abreviatura del código
const docInfo = IdentityDocumentTypeIdInfo[docType];
console.log(docInfo.description); // "Cédula de ciudadanía"
console.log(docInfo.abbreviation); // "CC"
```

<Callout type="info">
  Estas constantes están declaradas explícitamente y no realizan llamadas HTTP.
</Callout>

## Tipo de documento de factura (`BillDocumentType`)

```ts
import { BillDocumentType, BillDocumentTypeInfo } from "factus-js";

const code = BillDocumentType.ElectronicSalesInvoice;

console.log(code); // "01"
console.log(BillDocumentTypeInfo[code].description); // "Factura electronica de venta."
```

| Clave                          | Valor  | Descripción                                       |
| ------------------------------ | ------ | ------------------------------------------------- |
| `ElectronicSalesInvoice`       | `"01"` | Factura electrónica de venta.                     |
| `ElectronicTransmissionType03` | `"03"` | Instrumento electrónico de transmisión - tipo 03. |
