# Recepción RADIAN (`reception`)

El recurso `reception` gestiona las **facturas electrónicas recibidas** y los **eventos del ciclo de vida RADIAN** (acuse de recibo, reclamo, recepción de bienes, aceptación expresa y tácita). Se accede a través de `factus.reception`.

<Callout type="info">
  El flujo RADIAN tiene requisitos específicos de campos por tipo de evento.
  Para la referencia completa y autorizada, consulta la [documentación oficial
  de Factus](https://developers.factus.com.co/).
</Callout>

## Métodos disponibles

| Método                     | Verbo + Ruta                                                     | Descripción                                  |
| -------------------------- | ---------------------------------------------------------------- | -------------------------------------------- |
| `list(filters?)`           | `GET /v1/receptions/bills`                                       | Listar facturas recibidas                    |
| `get(id)`                  | `GET /v1/receptions/bills/{id}`                                  | Obtener detalle de una factura recibida      |
| `upload(input)`            | `POST /v1/receptions/upload`                                     | Registrar una factura recibida por CUFE/CUDE |
| `emitEvent(params, input)` | `POST /v1/receptions/bills/{bill_id}/radian/events/{event_type}` | Emitir un evento RADIAN                      |

---

## `list(filters?)`

Lista las facturas recibidas con filtros opcionales y paginación.

```ts
list(filters?: ReceptionBillFilters & { page?: number; per_page?: number }): Promise<ApiResponse<PaginatedData<ReceptionBill>>>
```

```ts
const response = await factus.reception.list({
  page: 1,
  per_page: 10,
});

const facturas = response.data.data;
const paginacion = response.data.pagination;
```

---

## `get(id)`

Obtiene el detalle completo de una factura recibida por su ID interno.

```ts
get(id: number): Promise<ApiResponse<UploadedReceptionBill>>
```

```ts
const response = await factus.reception.get(987);
console.log(response.data);
```

---

## `upload(input)`

Registra una nueva factura recibida usando su DIAN track ID (CUFE/CUDE).

```ts
upload(input: UploadReceptionBillInput): Promise<ApiResponse<UploadedReceptionBill>>
```

```ts
const response = await factus.reception.upload({
  track_id: "abc123def456...", // CUFE o CUDE de la factura
});

console.log(response.data);
```

---

## `emitEvent(params, input)`

Emite un evento del ciclo de vida RADIAN sobre una factura recibida.

```ts
emitEvent(params: EmitEventParams, input: EmitEventInput): Promise<ApiResponse<ReceptionBillEvent[]>>
```

```ts
import { EventCode, ClaimConceptCode } from "@factus-js/constants";

// Acuse de recibo (030)
const response = await factus.reception.emitEvent(
  {
    bill_id: 987,
    event_type: EventCode.RECEIPT_ACKNOWLEDGEMENT, // "030"
  },
  {
    // campos requeridos para el evento
  },
);

// Reclamo (031)
await factus.reception.emitEvent(
  {
    bill_id: 987,
    event_type: EventCode.INVOICE_CLAIM, // "031"
  },
  {
    claim_concept_code: ClaimConceptCode.DOCUMENT_INCONSISTENCIES,
  },
);
```

### Códigos de evento RADIAN (`EventCode`)

| Constante                  | Valor   | Descripción                   |
| -------------------------- | ------- | ----------------------------- |
| `RECEIPT_ACKNOWLEDGEMENT`  | `"030"` | Acuse de recibo de la factura |
| `INVOICE_CLAIM`            | `"031"` | Reclamo sobre la factura      |
| `GOODS_OR_SERVICE_RECEIPT` | `"032"` | Recibo de bienes o servicios  |
| `EXPRESS_ACCEPTANCE`       | `"033"` | Aceptación expresa            |
| `TACIT_ACCEPTANCE`         | `"034"` | Aceptación tácita             |

### Códigos de concepto de reclamo (`ClaimConceptCode`)

| Constante                     | Valor  | Descripción                     |
| ----------------------------- | ------ | ------------------------------- |
| `DOCUMENT_INCONSISTENCIES`    | `"01"` | Inconsistencias en el documento |
| `GOODS_NOT_DELIVERED_TOTAL`   | `"02"` | Bienes no entregados (total)    |
| `GOODS_NOT_DELIVERED_PARTIAL` | `"03"` | Bienes no entregados (parcial)  |
| `SERVICE_NOT_PROVIDED`        | `"04"` | Servicio no prestado            |

<Callout type="info">
  El flujo RADIAN estándar es: **030 → 032 → 033**. El evento 031 (reclamo)
  interrumpe el flujo. El evento 034 (aceptación tácita) se genera
  automáticamente por la DIAN tras 3 días hábiles si no hay reclamo.
</Callout>
