El paquete `@factus-js/types` expone todas las interfaces TypeScript utilizadas por el SDK: formas de respuesta de la API, tipos de entrada, primitivos compartidos y helpers geográficos. Si instalaste `factus-js`, ya están disponibles sin instalar nada más.

## Instalación

Si usas `factus-js`, los tipos ya están incluidos:

```ts
import type { ApiResponse, Customer } from "factus-js";
```

Si usas los paquetes internos directamente, instala `@factus-js/types` por separado:

<Tabs items={['npm', 'yarn', 'pnpm', 'bun']}>

<Tab value="npm">

```bash
npm install @factus-js/types
```

</Tab>

<Tab value="yarn">

```bash
yarn add @factus-js/types
```

</Tab>

<Tab value="pnpm">

```bash
pnpm add @factus-js/types
```

</Tab>

<Tab value="bun">

```bash
bun add @factus-js/types
```

</Tab>

</Tabs>

## Respuestas de la API

Todos los métodos del SDK devuelven una de estas dos formas.

### `ApiResponse<T>`

Respuesta estándar para operaciones que devuelven un único recurso.

```ts
interface ApiResponse<T> {
  status: string;
  message: string;
  data: T;
}
```

```ts
const response = await factus.invoices.getByNumber("SETP990000001");
// response.data es de tipo ViewInvoiceData
```

### `PaginatedData<T>`

Envuelve colecciones paginadas. Se usa dentro de `ApiResponse<PaginatedData<T>>`.

```ts
interface PaginatedData<T> {
  data: T[];
  pagination: {
    total: number;
    per_page: number;
    current_page: number;
    last_page: number;
    from: number;
    to: number;
    links: PaginationLink[];
  };
}

interface PaginationLink {
  url: string | null;
  label: string | number;
  active: boolean;
  page?: number;
}
```

```ts
const response = await factus.invoices.list({ page: 1 });
const { data, pagination } = response.data;
console.log(pagination.total); // total de facturas
console.log(pagination.last_page); // última página disponible
```

### `ErrorResponse` y `ApiError`

Forma del cuerpo de error cuando la API responde con un código 4xx/5xx. El SDK lanza un `FactusError` con esta información adjunta.

```ts
interface ErrorResponse {
  status: string;
  errors: ApiError[];
}

interface ApiError {
  code: number;
  message: string;
  detail: string;
  api_version: string;
}
```

```ts
import { FactusError } from "factus-js";

try {
  await factus.invoices.create({
    /* ... */
  });
} catch (err) {
  if (err instanceof FactusError) {
    console.log(err.statusCode); // HTTP status code
    console.log(err.errors); // ApiError[]
  }
}
```

<Callout type="info">
  Consulta la guía de [manejo de errores](/docs/error-handling) para más
  detalles sobre `FactusError`.
</Callout>

## Primitivos compartidos

Pares código/nombre que aparecen repetidamente en las respuestas de la API.

### `CodeNameObject`

Par `{ code, name }` — usado en `payment_form`, `payment_method`, `operation_type`, `document`, etc.

```ts
interface CodeNameObject {
  code: string;
  name: string;
}
```

### `CodeNameIdObject`

Triple `{ id, code, name }` — usado en `unit_measure`, `standard_code`, `tribute`, `legal_organization`, etc.

```ts
interface CodeNameIdObject {
  id: number;
  code: string;
  name: string;
}
```

## Tipos geográficos

### `Department`

Departamento tal como aparece anidado dentro de los objetos de municipio en respuestas de establecimiento.

```ts
interface Department {
  id: number;
  code: string;
  name: string;
}
```

### `MunicipalityObject`

Municipio completo (con departamento anidado) devuelto en respuestas de establecimientos.

```ts
interface MunicipalityObject {
  id: number;
  code: string;
  name: string;
  department: Department;
}
```

### `MunicipalityRef`

Referencia ligera de municipio (sin departamento) que aparece dentro de objetos de cliente/proveedor.

```ts
interface MunicipalityRef {
  id: number;
  code: string;
  name: string;
}
```

## Tipos de cliente

### `Customer`

Datos del adquirente tal como se almacenan en las respuestas de detalle de documentos.

```ts
interface Customer {
  identification: string;
  dv?: string;
  company?: string;
  trade_name?: string;
  names: string;
  address: string;
  email: string;
  phone: string;
  legal_organization_id: string;
  tribute_id: string;
  identification_document_id: string;
  municipality_id: string;
}
```

<Callout type="info">
  Para buscar un cliente existente por número de identificación antes de crear
  una factura, usa [`factus.catalog.getAcquirer()`](/docs/modules/catalog).
</Callout>

## Helpers de entrada

### `BillingPeriod`

Periodo de facturación opcional en facturas y notas crédito.

```ts
interface BillingPeriod {
  start_date: string; // "YYYY-MM-DD"
  start_time?: string; // "HH:mm:ss"
  end_date: string;
  end_time?: string;
}
```

### `EstablishmentInput`

Datos del establecimiento emisor. Se envía en `CreateInvoiceInput` y `CreateCreditNoteInput`.

```ts
interface EstablishmentInput {
  name: string;
  address: string;
  phone_number: string;
  email: string;
  municipality_id: string; // e.g. "980"
}
```

## Descarga y correo

Tipos compartidos por todos los módulos de documentos (facturas, notas crédito, documentos soporte, notas de ajuste).

interface DownloadXmlData {

```ts
interface DownloadPdfData {
  file_name: string;

```

### `SendEmailInput` y `SendEmailResponse`

```ts
interface SendEmailInput {
  email: string;
  pdf_base_64_encoded?: string;

interface SendEmailResponse {
  status: string;
  message: string;
}
```

### `DeleteResponse`

```ts
interface DeleteResponse {
  status: string;

}
```

## Retenciones

### `ItemWithholdingTax` y `DocumentWithholdingTax`

Retenciones a nivel de ítem y de documento, presentes en las respuestas de detalle.

```ts
interface ItemWithholdingTaxRate {
  code: string;
  name: string;
  rate: string;
}

interface ItemWithholdingTax {
  tribute_code: string;
  name: string;
  value: string;
  rates: ItemWithholdingTaxRate[];
}

interface DocumentWithholdingTax {
  tribute_code: string;
  name: string;
  value: string;
}
```

## Otros tipos compartidos

### `CompanyInfo`

Bloque de datos de la empresa emisora en las respuestas de detalle de documentos.

```ts
interface CompanyInfo {
  url_logo: string;
  nit: string;
  dv: string;
  company: string;
  name: string;
  graphic_representation_name?: string;
  registration_code: string;
  economic_activity: string;
  phone: string;
  email: string;
  direction: string;
  municipality: string;
}
```

### `NumberingRangeInfo`

Información del rango de numeración asociado al documento.

```ts
interface NumberingRangeInfo {
  prefix: string;
  from: number;
  to: number;
  resolution_number: string;
  start_date: string;
  end_date: string;
  months: number;
}
```

### `AllowanceChargeResponse`

Cargo o descuento tal como aparece en las respuestas de detalle.

```ts
interface AllowanceChargeResponse {
  concept_type: CodeNameObject;
  is_surcharge: boolean;
  reason: string;
  base_amount: string;
  percentage: string;
  amount: string;
}
```
